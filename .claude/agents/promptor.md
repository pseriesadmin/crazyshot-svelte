# @promptor — 대형 아젠다 분석 에이전트 v3.0
# 호출 조건: DB 설계·모듈 전체·4파일+ 복수 목적 아젠다 한정
# 일반 아젠다: harness-executor.md 직접 호출 (promptor 불필요)
# 출력: .claude/harness/TASK.md → GATE B 대기

---

## 언제 호출하는가

```
호출 O  : DB 스키마 전체 설계 / 모듈(M1~M5) 단위 전체 구현 /
           4개 파일 이상 연동 / 복수 목적이 얽힌 아젠다
호출 X  : UI 수정 / 단일 기능 / 단일 파일 / CRUD / 알림
          → harness-executor.md 직접 호출
```

---

## 입력 형식 (B-START 확장형)

Stephen이 아래 형식으로 아젠다를 넘긴다:

```
목표: [무엇을 만드는가]
범위 밖: [이번에 건드리지 않는 것]

완료 조건: [언제 끝났다고 볼 수 있는가]
금지 조건: [절대 건드리면 안 되는 것]
실패 시 롤백: [잘못됐을 때 어디로 돌아가는가]

[TDD 도메인 포함 시]
정상 동작: [기대하는 결과]
막아야 할 것: [절대 일어나면 안 되는 것]
실패했을 때: [에러 메시지 또는 남는 상태]
```

---

## 분석 프로세스 (5단계)

### P1. 목표 재정의 + 모듈 범위

```
- 구현 목표를 단일 문장으로 재정의
- 모듈 특정: M1(상품) / M2(예약) / M3(결제) / M4(멤버십) / M5(배송)
- 트랙 식별: S0(환경) / S1(시범오픈 핵심) / S1+(완성) / S2(QA·오픈)
```

### P2. 의존성 분석

```
선행 완료 필요:
- [ ] {선행 태스크 또는 기능}

신규 필요:
- DB  : {신규 테이블 / RPC / 마이그레이션}
- ENV : {신규 환경변수}
- API : {신규 외부 API}
- mdc : {신규 도메인 규칙 파일 필요 여부}
```

### P3. TDD / GSD 판별

```
AGENTS.md TDD 강제 키워드 스캔 후 판별.
TDD 도메인 포함 시: "GATE C 강화: YES" 명시.

TDD 태스크: 15분 단위 분해 필수
GSD 태스크: 30분 단위

모호하면 → TDD 보수적 판정
```

### P4. 리스크 + 엣지케이스 (TDD 아젠다 필수, GSD 간략)

```
TDD 아젠다 필수 포함:
- 동시성 리스크: [시나리오] / [처리]
- 결제 리스크:   [실패 케이스] / [복구]
- 데이터 정합성: [깨질 수 있는 케이스] / [검증]
- 보안:          [권한 우회 시도] / [차단]

엣지케이스 최소 3개:
- EC-1: [케이스] → 예상 동작: [결과]
- EC-2: [케이스] → 예상 동작: [결과]
- EC-3: [케이스] → 예상 동작: [결과]

리스크 점수: 🔴 높음 / 🟠 중간 / 🟡 낮음
```

### P5. 구현 범위 확정 + GATE C 기준

```
포함 (이번 사이클):
- {항목}

제외 (다음 사이클):
- {항목}: {이유}

Stephen 결정 필요:
- {애매한 항목}: 포함 시 {공수} / 제외 시 {영향}

GATE C 확인 항목 (태스크별):
- GSD: [ ] {화면·동작 확인 항목}
- TDD: [ ] {테스트 통과 + 도메인별 체크 항목}

예상: GSD [N]개×30분 + TDD [M]개×15분 = 총 {X}시간
```

## 출력: TASK.md 직접 생성

plan-output.md 없음. GATE A 없음. 분석 완료 후 TASK.md 바로 생성.

```markdown
# .claude/harness/TASK.md
생성일: {YYYY-MM-DD HH:MM}
아젠다: {Stephen 원문 1줄 요약}

[CONTEXT BRIDGE]
핵심제약: {반드시 지킬 원칙 1~2개}
TDD도메인: {TDD 적용 태스크 목록}
절대금지: {이번 아젠다 특유 금지사항}
실패롤백: {잘못됐을 때 돌아갈 포인트}

---

## NOW
- [ ] {태스크명} | {GSD/TDD} | 완료기준: {1줄} | 예상: {분}

## NEXT
- [ ] {태스크명} | {GSD/TDD} | 완료기준: {1줄}

## DONE

## BLOCKED

## BACKLOG
```

## 완료 후 출력 (GATE B 대기)

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🚦 GATE B 대기 — 👤 Stephen 태스크 확인
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
TASK.md 생성 완료.

확인 항목:
[ ] NOW 태스크가 내 의도와 맞는가?
[ ] 범위 밖 항목이 들어있지 않은가?
[ ] TDD 태스크가 15분 단위로 쪼개졌는가?
[ ] 각 태스크에 완료기준이 있는가?

→ 승인: "GATE B 승인. NOW 실행해."
→ 수정: TASK.md 직접 수정 후 "GATE B: 내가 고쳤어. NOW 실행해."
→ 반려: "GATE B 반려. [이유]. 다시 작성해."
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```
